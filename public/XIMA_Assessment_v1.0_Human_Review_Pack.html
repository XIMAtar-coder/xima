2. Algorithm & Scoring

2.1 Pillar Mapping

Questions are mapped to pillars via a cyclic formula:

getPillarForQuestion(n) → PILLAR_CYCLE[(n-1) % 5]

PILLAR_CYCLE = [computational_power, communication, knowledge, creativity, drive]
PillarQuestion IDsCount
Computational Powerq1, q6, q11, q16, q215
Communicationq2, q7, q12, q174
Knowledgeq3, q8, q13, q184
Creativityq4, q9, q14, q194
Driveq5, q10, q15, q204

2.2 MC Scoring

Each option has a score_value: 0, 1, 2, or 3.
pillar_score = mean(selected_score_values for that pillar's questions) / 3 × 100

Range: 0–100

2.3 Open Answer Blending

blended[pillar] = base[pillar] × 0.90 + open_contribution[pillar] × 0.10

open1 → 60% Creativity + 40% Communication (max 10% influence)
open2 → 60% Drive + 40% Knowledge (max 10% influence)

Computational Power is NOT affected by open answers (100% MC).

2.4 Drive: Baseline MC vs. True Longitudinal

MC Baseline (Assessment): Behavioral proxy measuring learning strategy under friction. Score calculated same as other pillars.

True Drive (Longitudinal):

Table: pillar_progress_snapshots
Sources: assessment_initial, mentor_session_completed, challenge_completed, assessment_retest

1. Identify 2 weakest pillars at baseline (earliest snapshot)
2. delta = latest_score - baseline_score for each weak pillar
3. velocity = delta / days_elapsed (min 1 day)
4. avg_velocity = mean(velocities)
5. score = max(0, min(100, round(50 + (avg_velocity / 0.20) × 25)))

Target velocity: 0.20 pts/day
State until ≥2 snapshots: "refining"
Cached in: profiles.drive_score, profiles.drive_velocity, profiles.drive_updated_at

2.5 XIMAtar Assignment

Guest mode: Top-2 pillar heuristic with hardcoded if/else chain.
Authenticated mode: Euclidean distance to 12 canonical profiles defined in ximatarTaxonomy.ts.
