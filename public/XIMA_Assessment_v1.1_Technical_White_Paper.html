1. Executive Technical Overview

The XIMA Assessment v1.1 is a scenario-based professional profiling instrument designed to generate a five-dimensional competency vector for each respondent. It is intended for use as a decision-support tool in hiring, coaching, and professional development contexts. It does not produce automated decisions.

Purpose

The instrument measures professional cognitive dispositions across five construct pillars through contextualised scenario questions that embed meaningful trade-offs. It is explicitly designed to surface growth leverage points (weakest pillars with highest development potential) rather than rank-order candidates.

Target Use Cases

  • Decision support: Providing structured signals to complement interviews and reference checks in hiring processes.
  • Development insight: Identifying specific professional growth areas for coaching and mentorship conversations.
  • Team composition analysis: Understanding pillar distribution within teams to identify capability gaps.

Architecture Overview

The assessment operates as a client-side application with the following components:

  • Trilingual i18n runtime: All question content is stored in locale files (EN/IT/ES) and served dynamically based on user language preference.
  • Cyclic pillar mapping: A deterministic function maps each question number to its pillar, ensuring consistent scoring regardless of locale.
  • Scoring engine: Computes pillar-level scores from MC responses (90% weight) and optional open-answer AI evaluation (10% weight).
  • XIMAtar assignment: Matches the respondent's pillar vector to the closest of 12 canonical archetype prototypes via Euclidean distance.
  • Freeze guard: Runtime hash verification ensures content integrity against the frozen v1.1 baseline.

2. Construct Framework

2.1 The Five-Pillar Model

Each pillar represents a distinct professional cognitive dimension. The definitions below specify both what the construct measures and what it explicitly does not measure.

Communication Ability to adapt messaging to diverse audiences, navigate interpersonal dynamics, and achieve alignment through structured dialogue. Linguistic fluency, extroversion, or social desirability. q2, q7, q12, q17
Knowledge Capacity for evidence evaluation, belief updating under uncertainty, and synthesis of information from multiple sources. Domain-specific expertise or factual recall. q3, q8, q13, q18

4. Scoring Model (Technical)

4.1 Item Scoring

Each MC question presents four options. Options are scored as ordinal values:

Option IndexScore Value
Option A (index 0)0
Option B (index 1)1
Option C (index 2)2
Option D (index 3)3
Design note: Options are not ranked by quality. The ordinal mapping captures response patterns for pillar vector construction. All options are designed to be equally defensible professional approaches (Cognitive Tension principle).

Aggregation Per Pillar

For each pillar, the raw score is the mean of all item scores assigned to that pillar:

raw_pillar_score = Σ(item_scores) / count(items_in_pillar)

4.2 Normalization

Raw scores (range 0–3) are normalized to a 0–100 scale:

normalized_score = (raw_pillar_score / 3) × 100

This produces a continuous score where 0 represents consistent selection of Option A and 100 represents consistent selection of Option D across all items in that pillar.

4.3 Open-Answer Integration

When active, open-answer responses are evaluated by an AI scoring engine and contribute a 10% weight to specific pillars:

Open QuestionBlends Into PillarsWeight
open1 (success story)Creativity, Communication10% of final pillar score
open2 (growth area)Drive, Knowledge10% of final pillar score
final_pillar_score = (0.90 × mc_normalized) + (0.10 × open_score_normalized)

Guardrails against bias: The AI scoring rubric uses structured criteria (specificity, measurability, self-awareness) rather than content-based judgments. Responses are evaluated for depth of reflection, not correctness of position.

4.4 Drive "Growth Velocity" Model

Drive is uniquely constrained to measure learning behaviour under friction rather than static motivation. All Drive items (q5, q10, q15, q20) present scenarios where:

  • The respondent has an acknowledged skill gap or repeated failure
  • Options encode different corrective strategies (feedback-seeking, deliberate practice, method-switching, structured self-critique)
  • No option represents "try harder" or generic persistence
Growth Velocity Definition: The rate at which an individual identifies friction in their own performance, designs a specific corrective strategy, and sustains deliberate practice toward improvement. This is measured through scenario responses, not self-report.

Euclidean Distance Calculation

d(Vcandidate, Vprototype) = √( (Dc-Dp)² + (CPc-CPp)² + (Cmc-Cmp)² + (Crc-Crp)² + (Kc-Kp)² )

The XIMAtar with the smallest distance is assigned as the respondent's archetype.

Tie Resolution

In the event of equal distances, the prototype appearing first in the canonical list (lion, owl, dolphin, fox, bear, bee, wolf, cat, parrot, elephant, horse, chameleon) is selected. This ensures deterministic, reproducible assignment.

Data Stored

The following are persisted to the database upon completion:

  • Pillar vector (5 normalized scores)
  • Assigned XIMAtar ID
  • Individual answer values per question
  • Open-answer text and AI rubric scores (if applicable)
  • Assessment metadata (field, language, timestamp, attempt ID)

7. Validity Claims & Non-Claims

7.1 Claims

XIMA Assessment v1.1 can reasonably claim the following based on its design and synthetic validation:

  • Construct separation: The five pillars are designed as distinct constructs with maximum observed cross-pillar correlation of r=0.34.
  • Scenario-based cognitive tension: All v1.1 items present meaningful trade-offs between two or more legitimate professional strategies, reducing response gaming.
  • Reduced social desirability bias (design intent): The Weakness-First methodology and anti-virtue-signaling constraints are systematically applied to all items.
  • Internal consistency: Mean Cronbach's α across fields ranges 0.63–0.67 in synthetic validation, indicating acceptable reliability for a development-oriented instrument.
  • Trilingual parity: Full content equivalence across EN, IT, and ES with runtime hash enforcement.
  • Deterministic assignment: XIMAtar archetype assignment is fully reproducible given the same input vector.

7.2 Non-Claims

XIMA Assessment v1.1 does NOT claim:

  • It is not a clinical diagnostic tool and should not be used for clinical assessment purposes.
  • It is not a measure of IQ or general cognitive ability.
  • It is not a personality inventory (e.g., Big Five, MBTI equivalent).
  • It is not predictive of job performance without contextual validation within the specific role and organization.
  • It is not free from all bias. Cultural, linguistic, and response-style biases may exist despite mitigation efforts.
  • It is not a standalone hiring decision system. It is designed exclusively as a decision-support signal to complement structured interviews, reference checks, and other assessment methods.
  • Synthetic validation data does not replace real-participant norming. Real-data milestones are set at N≥300 (correlation validation) and N≥500 (IRT calibration).

8. Bias Mitigation & Ethical Design

Weakness-First Design Rationale

The v1.1 methodology prioritises identifying a respondent's two weakest pillars with the highest growth leverage. This inverts the typical assessment approach of ranking strengths, instead surfacing development opportunities that are actionable in coaching and mentorship contexts.

Anti-Virtue-Signaling Logic

All items are designed to avoid Social Desirability gradients. Specifically:

  • No option is an obviously "correct" answer
  • All options represent legitimate professional strategies
  • Options encode cognitive trade-offs (e.g., speed vs. thoroughness, autonomy vs. collaboration)
  • Drive questions explicitly avoid "try harder" or "grit" rhetoric in favour of specific corrective behaviours

Cross-Pillar Bleed Control

Items are reviewed to ensure each question loads primarily on its assigned pillar. Questions that showed significant cross-loading in validation were revised in the v1.1 cycle. The cyclic mapping provides structural independence between pillar measurements.

Known Limitations

  • Cultural bias: Scenario contexts are Western-professional-centric. Respondents from different cultural frameworks may interpret "appropriate" professional behaviour differently.
  • Language bias: Despite trilingual parity, translation inherently introduces nuance shifts. English is the source language.
  • Response-style bias: The forced-choice format may disadvantage respondents who prefer nuanced, context-dependent responses.
  • Self-selection bias: The voluntary nature of the assessment may skew toward respondents with higher engagement.

Recommended Monitoring Practices

  • Track pillar score distributions by demographic segments to identify systematic bias
  • Monitor XIMAtar assignment distributions across fields for unexpected clustering
  • Conduct periodic item-level differential functioning analysis once N≥500
  • Review open-answer AI scoring patterns for language-dependent bias

10. Interpretation Guide (For HR / Coaches)

10.1 Reading Pillar Scores

Score RangeInterpretation
75–100Strength zone: The respondent consistently selects approaches indicating strong capability in this dimension. This is likely a natural operating mode.
50–74Functional range: The respondent shows adequate capability. This dimension is unlikely to be a significant barrier or a standout strength.
25–49Growth zone: This dimension represents a potential development area. The Weakness-First methodology specifically targets these zones for coaching conversations.
0–24Attention zone: Consistently low scores may indicate a genuine capability gap or a mismatch between the respondent's context and the assessment scenarios. Investigate before concluding.

Interaction Effects

Pillar scores should not be interpreted in isolation. Key interaction patterns include:

  • High CP + Low Communication: May indicate strong analytical capability that is under-leveraged due to difficulty translating insights for stakeholders.
  • High Creativity + Low Drive: May indicate strong ideation that lacks follow-through or structured execution.
  • High Knowledge + Low Creativity: May indicate deep expertise that is applied conventionally rather than innovatively.
  • Balanced profile (all 50–70): Indicates a generalist disposition. The Chameleon archetype often reflects this pattern.

10.2 Avoiding Misuse

  • Do not over-interpret a single pillar score as definitive of capability.
  • Do not use the assessment as a sole elimination filter in hiring processes.
  • Do not compare scores across different professional fields (scenarios differ).
  • Always combine assessment signals with structured interviews, reference checks, and work samples.
  • Always consider the respondent's self-selected field and professional context when interpreting scores.

10.3 Development Use

Coaching Pathways

The assessment's primary value lies in structuring development conversations:

  • Identify the two lowest pillars — these represent the highest-leverage growth areas.
  • Review specific responses in those pillars to understand the respondent's current approach patterns.
  • Design targeted interventions — e.g., if Drive is low, focus on structured feedback-seeking and deliberate practice routines rather than generic "motivation" advice.

Team Composition Insight

Aggregate pillar profiles across a team to identify:

  • Capability gaps (no team member strong in a critical pillar)
  • Over-concentration (too many team members with the same XIMAtar archetype)
  • Complementary pairings (pairing a high-Communication member with a high-CP member for client-facing analytical work)

How Version Bump Must Occur

  1. Update ASSESSMENT_VERSION to the new version (e.g., "1.2") in freezeGuard.ts
  2. Run in browser console: import('./lib/assessment/freezeGuard').then(m => m.regenerateHashes())
  3. Copy the logged hashes into ASSESSMENT_FREEZE_HASHES in freezeGuard.ts
  4. Commit with message: chore: bump assessment freeze to v1.2
  5. Rerun the full validation pipeline (Cronbach's α, cross-pillar correlation, DIF analysis)
  6. Update the Human Review Pack and this Technical White Paper

Change Governance Policy

  • No content changes are permitted under v1.1. Any modification to question text, options, scoring, or pillar mapping requires a version bump.
  • Translation corrections (fixing typos without changing meaning) require a hash update and version note, but not a full version bump.
  • New languages can be added without affecting existing hashes, but must be included in the freeze guard scope.
  • Scoring model changes require a major version bump (e.g., v2.0) and full revalidation.
Real-Data Milestones: The following validation steps are scheduled pending sufficient real-participant data:
  • N ≥ 300: Cross-pillar correlation validation with real data
  • N ≥ 500: Item Response Theory (IRT) calibration and differential item functioning analysis
  • N ≥ 1000: Predictive validity study (assessment scores vs. subsequent performance data)
