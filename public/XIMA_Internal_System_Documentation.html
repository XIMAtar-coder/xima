1. Executive Overview

XIMA is a psychometric hiring intelligence platform that replaces CV-first screening with a 5-pillar behavioral assessment, producing anonymized candidate profiles ("XIMAtars") that companies evaluate through a progressive 3-level challenge pipeline. The platform measures Computational Power, Communication, Knowledge, Creativity, and Drive — generating development-oriented signals (not scores or grades) that help businesses make more informed hiring decisions while giving candidates actionable self-awareness.

Core Problem

Traditional hiring is biased toward credentials, self-presentation, and unstructured interviews. XIMA replaces this with a structured, psychometrically-validated assessment pipeline that surfaces cognitive patterns and behavioral tendencies that CVs cannot reveal.

Target Users

BusinessCompanies creating hiring goals, launching challenges, evaluating submissions via signal panelsLive

Core Differentiator

XIMAtar-based anonymized profiles eliminate name/photo/institution bias. The "Weakness-First" methodology prioritizes identifying growth leverage rather than ranking candidates. No auto-rejection — businesses always make the final decision with qualitative signals, not numerical scores.

3.3 Scoring Logic

Multiple Choice Scoring

Each MC answer maps to ordinal value: 0, 1, 2, or 3
Pillar raw score = mean(answer_values for pillar questions)
Pillar normalized score = (raw_score / 3) × 100    // Scale to 0-100

Open Answer Scoring

// Client-side heuristic (fallback):
score = length(0-20) + relevance(0-25) + structure(0-20) + specificity(0-20) + action(0-15)
Total: 0-100

// Server-side (primary): LLM evaluation via analyze-open-answer edge function
// Pre-LLM: Non-answer detection (regex patterns, word count < 5 → score 5)
// Post-LLM: Red flag caps (generic→40, off_topic→25, contradiction→35, copy_paste→30)
// Post-LLM: Word count < 30 → cap at 35

Blending Formula (90/10)

OPEN_WEIGHT = 0.10   // 10% influence

open1 contributes to: Creativity (60%) + Communication (40%)
open2 contributes to: Drive (60%) + Knowledge (40%)
Computational Power: MC-only (no open-answer influence — deliberate design choice)

blended_pillar = base_MC × 0.90 + open_contribution × 0.10
Result clamped to [0, 100]

// Guest users: simplified if/else heuristic → archetype
// Authenticated users: Euclidean distance to 12 canonical profiles

distance(candidate, archetype) = √(Σ(candidate_pillar_i - archetype_pillar_i)²)

// Closest archetype wins

Cognitive Tension Enforcement

All MC questions are designed with "Cognitive Tension" — meaningful trade-offs between two legitimate strategies. No answer option is universally "best." This is enforced at the question design level (frozen in v1.1), not at runtime.

Anti-Social-Desirability Mechanisms

  • MC options avoid "obviously correct" answers — all options represent valid professional approaches
  • Drive questions specifically avoid "grit" rhetoric and motivation self-reports
  • Open-answer evaluator penalizes generic/vague responses (red flag: "generic" → cap at 40)
  • Non-answer detection catches "I don't know" / "not sure" patterns (trilingual regex)

Model Dependence Risk

Medium-High. All AI features route through ai.gateway.lovable.dev using google/gemini-2.5-flash. If the gateway goes down or the model changes behavior:
  • Open-answer scoring has local fallback (scoreOpenResponse in openResponse.ts)
  • L2 signals have local heuristic fallback (computeLevel2Signals local function)
  • CV analysis, challenge generation, and chat have NO fallback — they fail with error response
  • No model versioning/pinning — model behavior changes could affect scoring consistency
